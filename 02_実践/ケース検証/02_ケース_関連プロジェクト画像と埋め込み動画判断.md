# 02_ケース_関連プロジェクト画像と埋め込み動画判断

## 1. 事実（案件文脈）

- 対象は `related-projects` カードの画像表示と、Aboutページの動画埋め込み。
- 相談は次の混在だった。
  - `related-projects__img` クラスが複数要素に付いていて問題ないか
  - `object-fit: cover` / `object-position` の意味と使い分け
  - `iframe` の埋め込み属性の意味
  - 画像や動画素材選定時の運用

## 2. 判断プロセス

### 候補A: 作品ごとに画像クラスを分ける

- 例: `related-projects__sofa-img` / `related-projects__keyboard-img`
- 判定: 不採用
- 理由: 役割ではなく中身名に寄るため、BEMの再利用性が落ちる。

### 候補B: 同じ役割なら同じ element class を使う

- 例: どちらも `related-projects__img`
- 判定: 採用
- 理由: カード内の「画像本体」という役割が同一なら同じ要素クラスでよい。差分は modifier 側で扱う。

### 候補C: トリミング位置調整を `transform` で行う

- 例: `transform: translateX(-50%)`
- 判定: 原則不採用
- 理由: レイアウト副作用が出やすい。画像の見せる位置調整は `object-position` を優先する。

### 候補D: `object-fit: cover` + `object-position` で調整する

- 判定: 採用
- 理由: 比率維持で枠を埋めながら切り取り位置を制御できる。カード画像の目的に合う。

## 3. 案件固有として残すもの（02側）

- SP/PCで別画像を使う場合、同一作品か差し替えかを先に決める。
- `related-projects` のようなカードUIでは、表示属性を揃える。
  - `width` / `height`
  - `loading="lazy"`
  - `decoding="async"`
- 作品ごとの微調整は modifier で局所化する。
  - 例: `.related-projects__img--sofa { object-position: left center; }`

### 動画素材検索の実務メモ（この案件）

- 先に「どの用途で使う動画か」を固定する。
  - About背景の雰囲気づけ
  - 制作工程のB-roll
  - サービス説明の補助
- 検索語は日本語だけでなく英語も併用する（素材サイトのヒット率が上がる）。
  - 例: `creative studio team b-roll horizontal`
  - 例: `designer workspace cinematic 16:9`
  - 例: `web design process close up no talking`
- 除外語でノイズを減らす。
  - 例: `-vertical -text -watermark -interview`
- 候補比較は次の順で行う。
  1. 画角がレイアウト比率に合うか（16:9 / 4:3）
  2. テキスト重ね時に視認性が落ちないか
  3. サイト全体のトーン（色温度・動き量）と合うか
  4. ライセンス条件が実運用に合うか

## 4. 基礎概念へ戻すもの（01側）

- 再利用可能な原理:
  - 同じ役割の繰り返し要素は同じ element class を使う。
  - `object-fit: cover` は比率維持で枠を埋め、はみ出しを切り取る。
  - 切り取り位置調整は `object-position` を優先し、位置移動系 `transform` と混同しない。
  - `iframe` は外部コンテンツを埋め込む要素で、`title` と `allowfullscreen` は意味上の最小セットとして扱う。
- 判断順:
  1. 同一役割か（クラス統一）
  2. 比率固定か（枠の `aspect-ratio`）
  3. 切り取り許容か（`cover` / `contain`）
  4. 位置調整は表示範囲か実体移動か（`object-position` / `transform`）
- 例外条件:
  - 作品ごとに切り取り要件が大きく異なる場合のみ modifier を追加する。
  - 切り取り許容不可（文字や顔が欠けるとNG）の素材は別画像運用を検討する。
