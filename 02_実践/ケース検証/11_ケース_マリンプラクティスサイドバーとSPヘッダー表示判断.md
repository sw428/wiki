# 11_ケース_マリンプラクティスサイドバーとSPヘッダー表示判断

## 1. 事実（案件文脈）

- 対象は、マリンプラクティス制作中のサイドバー、ヘッダー、SP表示まわりのHTML/CSS相談ログ。
- 同一回で次の議題が混在した。
  - `article + article` やバナー同士の縦間隔を、`gap` / `margin-top` / `margin-bottom` のどれで取るか
  - サイドバーのバナー画像の上に仮テキストを重ねる方法
  - SPヘッダーでハンバーガーメニューを写真の上ではなく青帯側に置く方法
  - `visible-sp` / `visible-pc` と、通常フロー・相対配置・絶対配置の使い分け
  - `button` の初期表示、flex item、`display: grid`、`place-content` の責務分離
  - `aspect-ratio` を当てた画像・文字入り画像のサイズ調整、`cover` / `contain` / `height: auto` の判断
- 案件本体の実践記録は `02_実践/ポートフォリオサイト/マリンプラクティス/01_実装で意識したこと.md` に残し、再利用できる原理だけを `01_基礎概念` へ戻す。

## 2. 判断プロセス

### ケース1: 反復要素の間隔をどう取るか

#### 候補A: 各要素へ `margin-bottom` を付ける

- 判定: 条件付き
- 理由: 単純だが、最後の要素にも外側余白が残る。次のブロックとの距離まで同じ責務で持ってしまいやすい。

#### 候補B: 親で縦並びを作り `gap` を使う

- 判定: 採用候補
- 理由: 同じ種類の要素が連続する一覧なら、間隔は親の責務として扱いやすい。`gap` は要素間だけに入り、先頭・末尾には出ない。

```css
.article-list {
  display: grid;
  gap: 16px;
}
```

#### 候補C: 隣接兄弟で後続要素の上にだけ余白を付ける

- 判定: 採用候補
- 理由: 既存HTMLで親をFlex/Grid化しない場合や、単純な縦積みの一部だけ間隔を整えたい場合に読みやすい。

```css
.sidebar__banner + .sidebar__banner {
  margin-top: 10px;
}
```

#### ケース1の判断軸

- 同種反復を親で束ねられる: `gap`
- 既存構造を保ったまま、2個目以降にだけ距離を付けたい: `+` 隣接兄弟の `margin-top`
- 次の別ブロックとの距離を作りたい: ブロック側の `margin-bottom`
- 最後に余白が残ると困るなら、全要素への一律 `margin-bottom` は避ける

### ケース2: バナー画像の上に文字を重ねる方法

#### 採用形

```css
.sidebar__banner {
  position: relative;
}

.banner__text {
  position: absolute;
  inset: 0;
  display: grid;
  place-items: center;
}
```

#### 理由

画像の上に文字を重ねたい場合は、親バナーを基準箱にし、その中でテキストを絶対配置する。
`position: absolute` だけでは「どの箱を基準にするか」と「中でどう中央に置くか」が決まらないため、親の `position: relative`、子の `inset: 0`、子の内部整列をセットで見る。

#### 補足

- 親の高さは画像が通常フローで作る。
- テキストは絶対配置なので、親の高さを作る役割は持たない。
- `display: grid` / `place-items: center` は、絶対配置されたテキスト箱の中で文字を中央に置くための指定。

### ケース3: SPヘッダーのハンバーガーをどこに置くか

#### 候補A: 写真の上へ `position` で重ねる

- 判定: 不採用寄り
- 理由: 本当にMV画像の上へ重ねるデザインなら有効だが、今回の意図は「写真の上にかぶせる」ではなく、SP用の青帯メニューをMVの前に置くこと。

#### 候補B: SP専用のメニュー行を通常フローで置き、表示制御だけ `visible-sp` に任せる

- 判定: 採用
- 理由: 位置関係がHTMLの順番どおりになり、画像へ重なる副作用が出にくい。表示制御と配置責務を分けられる。

```html
<div class="site-header__menu visible-sp">
  <button class="hamburger-button" type="button" aria-label="メニューを開く">
    <span class="hamburger-button__line"></span>
    <span class="hamburger-button__line"></span>
    <span class="hamburger-button__line"></span>
  </button>
</div>

<div class="site-header__mv">
  <img class="site-header__mv-image" src="img/mv@2x.png" alt="" width="1300" height="346">
</div>
```

#### ケース3の判断軸

- SPでだけ存在する行: `visible-sp`
- PCでだけ存在するナビ: `visible-pc`
- 写真に重ねる必要がない: 通常フローで置く
- 本当に重ねるデザイン: `position` を使う
- SPでヘッダー全体に余白を入れたくない: `site-header` 全体ではなく、余白が必要な内側要素へ `padding-inline` を分ける

### ケース4: ハンバーガーボタンの箱と中の線を分ける

#### 判断

- 親の `align-items: center` は、ボタン自体を青帯の縦中央へ置く。
- `.hamburger-button` の `display: grid` は、ボタンの中の3基礎概念を並べる。
- `place-content: center` は、ボタン箱の中で3基礎概念を中央へ置く。
- `width` / `height` を書かなくても、線の幅・高さ・`gap` でボタンの自然サイズはできる。
- ただし実務では、押せる領域と線の見た目を分けるため、ボタン側に `width` / `height` を明示した方が安定する。

```css
.hamburger-button {
  appearance: none;
  display: grid;
  place-content: center;
  gap: 6px;
  width: 40px;
  height: 40px;
  margin: 0;
  padding: 0;
  border: 0;
  background: transparent;
  cursor: pointer;
}

.hamburger-button__line {
  display: block;
  width: 24px;
  height: 3px;
  background-color: var(--color-white);
}
```

#### 補足

親がFlexなら、ボタンは外側ではflex itemとして扱われる。
一方でボタン自身の `display: grid` は、中の線の並べ方を決める。
つまり同じ要素でも「外側から見た役割」と「内側をどう並べるか」は別で考える。

### ケース5: 文字入り画像・ボタン画像をどう小さく見せるか

#### 判断

- 写真カードのように枠いっぱいへ見せたい: `aspect-ratio` + `object-fit: cover`
- 全体を切らずに見せたい: `object-fit: contain`
- 電話画像・予約画像・ロゴのように文字が切れると困る画像: `width` + `height: auto`
- 位置だけ調整したい: `object-position`
- 箱そのものを動かしたい: `margin` / `transform`

#### 理由

`aspect-ratio` は「比率の箱」を作る指定であり、画像そのものを小さく見せる指定ではない。
文字入り画像を比率枠に押し込むと、`cover` で切れたり、`contain` で余白が出たりする。
単に比率を保って縮小したいだけなら、画像に `width` を決めて `height: auto` にする方が自然。

## 3. 案件固有として残すもの（02側）

- マリンプラクティスのサイドバーバナーは、画像を通常フローに残し、その上へ仮テキストを絶対配置で重ねる。
- バナー同士の間隔は、親で一覧化できる場合は `gap`、既存構造を保つ場合は `.sidebar__banner + .sidebar__banner` のような隣接兄弟で取る。
- SPヘッダーのハンバーガーは、写真へ重ねるのではなく、SP専用メニュー行として通常フローに置く。
- SPでヘッダー全体に余白を入れたくない場合は、`site-header` 全体ではなく、余白が必要な内側要素へ分ける。
- 予約・電話などの文字入り画像は、切り抜き用の `aspect-ratio` 枠に押し込まず、必要に応じて `width` + `height: auto` で扱う。

## 4. 基礎概念へ戻すもの（01側）

- 再利用可能な原理:
  - 同種反復の間隔は `gap`、既存構造内の2個目以降だけなら隣接兄弟の `margin-top`、別ブロックとの距離は `margin-bottom` と分ける。
  - 画像上に文字を重ねる場合は、基準箱、重ねる箱、中央寄せの責務を分ける。
  - SP専用要素は表示条件と配置責務を分け、重ねる必要がないなら通常フローで置く。
  - flex itemとしての外側の役割と、`display: grid` による内側の並べ方は別で見る。
  - `place-content` は親帯の中央寄せではなく、ボタン箱内の中身配置を担当する。
  - `aspect-ratio` は比率枠を作る指定であり、画像を小さくする指定ではない。
  - 文字入り画像は、切り抜きよりも比率維持表示を優先する。
- 判断順:
  1. 反復要素の間隔か、別ブロックとの距離かを分ける
  2. 画像上へ重ねたいのか、通常フローで上下に置きたいのかを分ける
  3. 親の中央寄せか、子の内部中央寄せかを分ける
  4. 画像を切ってよいか、全体を見せるべきかを分ける
- 例外条件:
  - デザイン上、MV画像の上にメニューを重ねる必要がある場合は、通常フローではなく `position` を使う。
  - 写真カードのように多少切れても見栄えを優先する場合は、`contain` ではなく `cover` を選ぶ。
