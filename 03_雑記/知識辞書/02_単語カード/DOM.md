# DOM

## 読み方

- ドム

## 定義

- HTMLをブラウザが扱いやすい形に変換した、ページ構造のデータ。
- HTMLファイルそのものではなく、ブラウザ内部で作られる構造ツリー。

## ここを見る理由

- JavaScriptが何を操作しているかを理解するため。
- CSSが見た目を決める前に、ページの構造側で何が存在しているかを分けて見るため。
- 要素ノード、属性、テキストノードを混同しないため。

## 一瞬イメージ

HTMLが文章だとしたら、DOMはその文章をブラウザが分解して作った構造図。

```txt
HTML
-> ブラウザが読む
-> DOMになる
```

```html
<p>こんにちは</p>
```

ブラウザ内部では、ざっくり次のような構造として扱われる。

```txt
p要素
└ テキストノード「こんにちは」
```

## 重要ポイント

- DOMはHTMLから作られる。
- 要素、属性、テキストなどをツリー構造で持つ。
- JavaScriptで取得・変更できる。
- CSSが見た目を決める前の「構造側」の情報として見る。
- DOMとCSSOMを合わせて、画面に出すためのレンダリング処理へ進む。
