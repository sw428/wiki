# wiki

このリポジトリは、Web制作・フロントエンド学習・実装中の判断を整理するための個人Wikiです。

HTML / CSS / JavaScript の学習メモ、制作中に出た疑問、検証結果、案件やポートフォリオ制作で得た判断基準を、あとから再利用しやすい形で残しています。

このWikiは、完成された知識辞書ではなく、実装・質問・検証・修正を通して理解を育てていく外部脳として運用します。

このWikiは、公開アプリとして管理する前提ではなく、ローカルMarkdownを中心に使います。  
記憶力や整理能力が高くなくても、必要な判断へ戻り、社会で求められる仕事に対して最低限のパフォーマンスを出すための補助として扱います。

現在は、web.dev、MDN、CodeJump Pro、ポートフォリオ制作を中心に、HTML / CSS / JavaScript の基礎と実務判断を整理しています。

## このWikiで扱う内容

- HTML / CSS / JavaScript の基礎概念
- Web制作中に出た疑問と検証結果
- ポートフォリオ制作・実践で得た判断基準
- 実装時につまずいた箇所と、あとから修正された認識
- GPT / Codex との対話を通して整理した学習ログ

## 運用方針

このWikiでは、知識を単発で並べるのではなく、実装中の違和感、検証履歴、理解が進んだ説明、あとから修正された認識を残します。

目的は暗記ではなく、思考の可視化と判断基準の形成です。
最初から体系化しきることより、実装・質問・検証・修正を通して、あとから戻れる知識ベースに育てることを優先します。
きれいに増やすことより、迷ったときの入口を減らし、確認順と判断軸を絞ることを優先します。

## GPT / Codex / 自分の役割

ChatGPTは思考整理と方針確認、Codexは整理・統合・反映、自分は最終判断を担当します。

対話と実装を往復しながら、つまずきやすい概念、理解が進んだ説明、あとから修正された認識を記録していきます。

## 対外利用時の前提とレビュー

このWikiは、対話・実装・検証の過程で得た理解を、あとから読み直し、学習や実務に活かせる形へ整理していくための場所です。

ここに残している内容には、独自の表現や思考ログも含まれます。それらは、概念を扱いやすくしたり、判断の過程を追いやすくしたりするための補助線として扱います。

内容は、学習経験・作業環境・理解の進み方に個人差があることを前提に、実装力と思考の筋道を整えていくための通過点です。実務利用・対外的な説明に使う前には、必要に応じて、実務経験者や専門知識を持つ人のレビューを通し、より読みやすく、伝わりやすい表現へ整えることも同時に必要としています。

レビューは、仕上げの段階だけでなく、日々の対話・実装・ツール利用の中で認識を更新し続ける運用でもあります。ツールの出力を確認し、その結果に応じてプロンプトや任せる範囲を調整し、さらに出力を見直すことで、Wiki全体の一貫性と再利用性を高めていきます。

このWikiの運用システム自体は、著者が一人で組み立てたものです。ただし、読む側もここにある情報だけで完結させず、実績のある人や専門知識を持つ人のレビューを通すことで、著者と読者の双方にある知識や経験の凹凸がさらに整い、学習がより良い方向につながると考えています。

端的に言えば、このWikiは、生活を成り立たせるための実務力を育てながら、自分の知的欲求や好奇心を満たし、周囲への配慮も含めて学びを整理していくための場所です。

## 詳細な運用思想（内部運用寄り）

ここから先は、このWikiをどう育てているかの運用思想です。
初めて読む人は、まず「このWikiで扱う内容」「運用方針」「フォルダ構成」だけを見れば十分です。

### 対話で育てる外部脳Wiki

このWikiは、最初から完成された知識辞書を作るための場所ではありません。  
第一段階の目的は、実装中の違和感・検証履歴・主観的理解・GPTとの照合結果を残し、あとから認識の変化を追えるようにすることです。

知識を単発で並べるのではなく、次の点を見える形にします。

- どの概念で何度もつまずいたか
- どの説明で理解が進んだか
- どの認識があとから修正されたか
- 実装上どこに重みを置くべきか
- 自分がどの順番で問題を切り分けているか

このため、このWikiでは確定知識だけでなく、検証途中の仮説・主観・修正履歴も扱います。

### 段階運用

このWikiは「正しい知識だけを並べる保管庫」ではなく、実装・検証・質問・修正を通して作る認識履歴でもあります。  
最初から体系化しきることを目的にせず、まずは実際に繰り返し出た違和感と検証を記録します。

運用は次の段階で進めます。

1. 実装中の違和感・仮説・GPT照合を残します
2. 何度も出る概念を見つけます
3. 重要度の高い概念に重みをつけます
4. 用語カードや判断基準に再整理します
5. 実装時に戻れる知識ベースにします

「知識を整理する前に、まず検証回数と認識の変化を残す」ことを優先します。

### 明文化しすぎない設計

このWikiは、ユーザー / ChatGPT / Codex が意思疎通しながら認識をすり合わせるための中間媒体でもあります。
そのため、すべての判断基準を各ファイルへ直接書き切る運用にはしません。

直接記述を増やしすぎると、ルールが固定化され、文脈ごとの調整余地が狭くなります。
本来は会話・検証・実装の流れで調整すべき判断まで硬直化しやすくなるため、一部の認識は「会話や検証履歴の流れで共有する前提」として扱います。

これは記述漏れではなく、調整幅を残すための意図的な設計です。

### 三者の性能を制限しないシステム運用

このWikiは、ユーザーがChatGPTやCodexの出力をすべて事前に理解し、完全に制御することを前提にしません。

ChatGPTやCodexには、ユーザー個人が把握しきれない知識の包括性・処理能力・整理能力があります。  
そのため、AIをユーザーの現在の理解範囲へ無理に縮小させるのではなく、それぞれが最大限の性能を発揮できる余地を残します。

ただし、AIへ判断を丸投げするわけではありません。  
出力・整理・反映の結果は、最終的にユーザーが確認し、必要に応じて修正します。
ChatGPTやCodexには、ユーザーへ過度に迎合させず、認識のズレ・不足・別視点を提示できる状態を維持します。

この運用では、ユーザーの知識を単純に足し算で増やすだけではなく、ユーザー・ChatGPT・Codexの三者の役割を掛け合わせます。  
ユーザーは最終判断、ChatGPTは思考整理と方針確認、Codexは整理・統合・反映を担当します。  
この三者間の対話と検証を繰り返すことで、判断精度・再利用性・整理能力が相互作用的に伸びる設計にしています。

---

### 三者運用の有効性と不完全性

ただし、ChatGPT / Codex / 自分の三者運用だけで、現在の社会システムに十分適合できるとは考えない。

三者運用は、理解の整理、判断基準の形成、実装中の迷いの可視化には有効である。
しかし、良くも悪くも、ユーザー・ChatGPT・Codexの三者だけで構成された判断は不完全であることを前提にする必要がある。

ユーザーには、経験・体調・環境・知識量による認識の偏りがある。
ChatGPTやCodexにも、文脈の取り違え、過剰な一般化、実務現場の温度感とのズレ、出力のもっともらしさによる誤認などの限界がある。

そのため、三者運用によって整理された内容は、最終的な正解として固定するのではなく、検証と更新の対象として扱う。

現在の社会は、個人の内側の理解だけで成り立っているのではなく、採用、実務、納品、評価、役割分担、責任範囲などの合理的な仕組みによって動いている。
そのため、自分の理解や整理方法を、現在成り立っている社会システムに接続していくには、実際にその中で評価を受け、仕事を進めている上位者・実務経験者・採用側・現場担当者のレビューが必要である。

このレビューは、精神論や結果論として受けるものではない。
現在の社会で通用している自然摂理的、合理的、判断基準とのズレを確認し、成果物、説明、優先順位、任せる範囲を現実の仕事に合わせて調整するための工程である。

このWikiは、内側の理解を整理するための外部記憶であり、三者運用の有効性と不完全性の両方を前提にしながら、最終的には、実際に社会で運用されている合理的な評価・実務システムへすり合わせていくことを目的とする。

## 用語の扱い

このWikiには、ChatGPTなどのLLMとの対話を通じて整理した性質上、
一般的な専門用語ではない表現や、個人的な造語が含まれる場合があります。

それらは、理解をつなぐための補助線であり、
必ずしも理屈やイメージそのものが誤っているという意味ではありません。

このWikiで造語や独自表現を使う目的は、
認知負荷を減らし、自分の中で概念を扱いやすくするためであります。

意思疎通を妨げることや、知識を誇示すること、混乱させること
他者に対して優位に立つことを目的としたものではありません。

公式用語・一般的な専門用語・実務上の標準表現とは分けて確認し、
最終的には個人の責任で使用してください。
