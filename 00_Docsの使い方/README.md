# 00_Docsの使い方

## この場所の目的

このプロジェクトの目的は、公式に正しい知識を確定保存することではない。  
目的は、現段階での認識を、できるだけ少ない視線移動で一つに統合し直せるようにすること。

## 最優先ルール

- 正しさの確定より、現段階での認識統合を優先する
- 既存優先。衝突したら既存の名前・置き場・流れを優先する
- 新しい分類やフォルダは極力増やさない
- ユーザーと GPT / Codex の両方が見る場所を最小化する
- 見返さない情報は残さないか、薄く残す
- 分からなくても、現時点の暫定認識として一つに寄せる

## 運用の基本

- 迷った内容はまず `03_雑記` に置く
- 本線化するのは、今後の思考の土台として見返すものだけ
- `05_保留` は最小限だけ使う
- `04_資料庫` も厳選し、再読しない資料は厚く保存しない
- 新規ファイルを増やす前に、既存ノートへの追記で済むかを優先して判断する
- GPT の出力は正答ではなく、認識を整えるための統合材料として扱う

## 判断に迷ったとき

最優先の問いはこれだけにする。  
「これは後で見返して思考の土台になるか？」

- yes → 既存の本線ノートへ追記、または `02_学習本線`
- not sure → `03_雑記`
- no → 残さないか、1行だけ残す
