# 02_型注釈と型推論

## 目的

- 明示注釈と推論の使い分けを固定する。

## ルール

- 明確な初期値があるときは推論を活かす
- 境界（関数引数・戻り値・外部データ）では注釈を明示する

## 要点

- 型注釈: `const id: string = "a1";`
- 型推論: `const id = "a1";` -> `string` と推論
- `any` は最後の手段として扱う

## 例

```ts
const title = "Hello"; // 推論: string

function formatPrice(price: number): string {
  return `${price}円`;
}
```

## 判断基準

1. 推論で十分か
2. 境界で型が曖昧になっていないか
3. `any` に逃げる前に型定義を足せるか
