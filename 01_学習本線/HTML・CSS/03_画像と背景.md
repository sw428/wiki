# 03_画像と背景

## 目的

- `<img>` と `background-image` の役割差を混同せずに判断できるようにする。
- 「背景が見えない」を、画像ファイルの問題ではなく `box` 面積の問題として切り分けられるようにする。
- 装飾画像の形式選定（`SVG` / `PNG`）を再利用しやすい判断軸で固定する。

## 最初に押さえる結論

1. `<img>` は文書内の意味を持つ画像要素（`alt` を持つ）。
2. `background-image` は既にある box への背景描画で、画像自体は box を作らない。
3. 背景が見えない時は、まず「背景を描く box に面積があるか」を確認する。

## 背景画像と描画ボックス

- `background-image` は、既存要素の `box` に背景として描画される。
- 背景画像は DOM に意味を追加しない。
- 背景画像は要素サイズ（面積）を作らない。
- そのため、要素側に `width` / `height` / `padding` / テキストなどの面積がないと見えにくい。

| 観点 | `<img>` | `background-image` |
|---|---|---|
| 役割 | 意味画像（文書内容） | 装飾背景（見た目） |
| DOM上の扱い | 要素として存在する | 新しい要素は増えない |
| セマンティクス | あり（`alt`） | なし |
| box面積への寄与 | 自身が描画対象になる | 背景画像だけでは面積を作らない |

## 背景が見えない時の確認順

1. 背景を描く要素に面積があるか（`width` / `height` / `padding` / テキスト）
2. `display: inline` の空要素になっていないか
3. 画像URLが正しいか
4. `background-size` / `background-position` が意図どおりか

補足:
- `display: block` / `inline-block` は面積を作りやすいが、高さ指定も内容もないと高さ `0` になり得る。

## 装飾背景の描画ルール

| プロパティ | 意味 |
|---|---|
| `background-size: cover` | 要素全体を覆う（比率維持、切り取りあり） |
| `background-size: contain` | 要素内に収める（比率維持、余白あり） |
| `background-position` | 切り取り時にどこを見せるかを決める |

## `<img>` と `background-image` の選択基準

- 意味画像として読ませる必要があるなら `<img>` を使う。
- 見た目の演出・装飾なら `background-image` を使う。
- 同じ見た目でも、意味の有無で選択を分ける。

補足:
- `object-fit` / `aspect-ratio` を使った「枠の安定化」は 07 側で扱う。

## `SVG` と `PNG` の使い分け（装飾画像）

- `SVG` が向く:
  - アイコン、ロゴ、単純図形、線画
  - 拡大縮小しても劣化しにくい素材
- `PNG` が向く:
  - 透過が必要なラスター画像
  - 写真的でないが、ピクセル表現を固定したい素材

補足:
- 写真は配信効率の観点で `WebP` / `AVIF` を優先検討し、`PNG` 常用は避ける。

## 判断基準

1. これは意味画像か、装飾背景か
2. 装飾背景なら、背景を描く box に面積があるか
3. 装飾画像の形式は `SVG` / `PNG` のどちらが適切か
4. 実装安定化（`object-fit` / `aspect-ratio` / CLS）は 07 で設計する
