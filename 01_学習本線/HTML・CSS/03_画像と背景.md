# 03_画像と背景

## 目的

- 背景描画（CSS）と画像要素（HTML）の役割差を混同しない。

## ルール

- `background-*` は視覚装飾（セマンティクスを持たない）。
- `<img>` は文書内の意味を持つ要素（`alt` 含む）。
- `display: inline` 要素は内容がなければ見える背景領域が作られにくい。
- `display: block` / `inline-block` はサイズ指定で領域を先に確保しやすい。

## 背景画像を見る前提

- `background-image` は「既にある要素の背景として描画」される。
- 背景画像そのものが要素サイズ（箱）を作るわけではない。
- そのため背景を安定して見せるには、要素側に `width` / `height` / `padding` / コンテンツなどで領域が必要。
- `display: inline` は行内内容量に依存して領域が決まりやすく、空要素では背景を見せる箱を作りにくい。

## `background-size` の基礎

| 値 | 挙動 |
|---|---|
| `cover` | 要素全体を覆う。比率維持のため一部トリミングが起こる。 |
| `contain` | 要素内に収める。比率維持のため余白が出ることがある。 |

補足: `fr` と違い、こちらは画像の描画ルール。レイアウト計算と混同しない。

## `object-fit` と `background-size` の違い

- `object-fit`: `<img>` / `<video>` のような置換要素の中身の描画ルール。
- `background-size`: 要素背景として置いた画像の描画ルール。
- どちらも `cover` / `contain` を使うが、対象（意味画像か装飾背景か）が違う。
- 判断基準:
  - 意味画像（`alt` が必要）なら `<img>` + `object-fit`
  - 装飾なら `background-image` + `background-size`

## `background-position` の基礎

- `background-position: 50% 50%` は画像中心を要素中心に合わせる。
- トリミング時に「どこを見せるか」を決めるプロパティとして使う。

## `SVG` と `PNG` の使い分け（実務）

- `SVG` が向く:
  - アイコン、ロゴ、単純図形、線画
  - 拡大縮小しても劣化しにくい素材
- `PNG` が向く:
  - 透過が必要なラスター画像
  - 写真的でないが、ピクセル表現を固定したい素材

補足:
- 写真は配信効率で `WebP` / `AVIF` を優先検討し、`PNG` 常用は避ける。
- `SVG` は `viewBox` を基準に作り、表示サイズはCSSで決める。
- `PNG` は表示寸法を先に決め、必要に応じて `2x` 書き出しを用意する。

## `<img>` は置換要素として扱う

- `<img>` は `display: inline` で始まるが、単なるテキストinlineではなく置換要素。
- 画像という外部内容を持つため、テキスト要素とは高さ・基準線・サイズ計算の見え方が違う。
- 下隙間や比率崩れを切り分ける時は、`display` だけでなく「置換要素かどうか」を先に確認する。

## 例

```css
.hero {
  background-image: url("hero.jpg");
  background-size: cover;
  background-position: center;
}
```

## 初期認識メモ（確度）

- 確度: 高
  - `cover` は「覆う」、`contain` は「収める」という整理。
  - `img` と `background-image` は意味レイヤーが違うという整理。
- 確度: 中
  - `img`/`input`/`video` などを「インライン例外」として把握した点（正確には置換要素として整理すると明瞭）。

## 判断基準

- 意味が必要なら `<img>`。
- 装飾なら `background-image`。
- トリミング制御が必要なら `background-size` + `background-position` をセットで考える。
