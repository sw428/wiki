# 09_セレクタと構造依存

## 目的

- セレクタが「何を数えているか」を、クラス名の見た目ではなくDOM構造から判断する。
- カード一覧や文章内要素の順番指定で、`:nth-child()` と `:nth-of-type()` を混同しない。

## 最初に押さえる結論

- `:nth-child()` は、親の子要素全体の中で何番目かを見る。
- `:nth-of-type()` は、同じタグ種類の兄弟要素の中で何番目かを見る。
- どちらも、クラスを持つ要素だけを集めて数えるわけではない。
- カード一覧の配置差分は、基本的に `:nth-child()` か明示クラスで扱う。

## `:nth-of-type()` の見方

```css
p:nth-of-type(2) {
  color: red;
}
```

```html
<section>
  <h2>タイトル</h2>
  <p>1つ目のp</p>
  <div>途中のdiv</div>
  <p>2つ目のp</p>
</section>
```

- この場合、赤くなるのは2つ目の `p`。
- `div` は `p` ではないため、`p` の順番には入らない。
- `:nth-of-type()` は「同じタグ種類の中で何番目か」を見る。

## `:nth-child()` の見方

```css
p:nth-child(2) {
  color: red;
}
```

```html
<section>
  <h2>タイトル</h2>
  <p>1つ目のp</p>
</section>
```

- この `p` は親の子要素全体の2番目なので当たる。

```html
<section>
  <h2>タイトル</h2>
  <div>説明</div>
  <p>1つ目のp</p>
</section>
```

- この `p` は親の子要素全体では3番目なので、`p:nth-child(2)` には当たらない。
- `:nth-child()` は「親の子要素全体の中で何番目か」を見る。

## クラス順では数えない

```css
.c-blog-card:nth-of-type(-n + 2) {
  grid-column: span 3;
}
```

これは「`.c-blog-card` の中で1〜2番目」ではない。

正確には、同じタグ種類の兄弟要素の中で1〜2番目で、さらに `.c-blog-card` を持つ要素に当たる。

```html
<div class="intro">説明</div>
<div class="c-blog-card">カード1</div>
<div class="c-blog-card">カード2</div>
<div class="c-blog-card">カード3</div>
```

- 1番目の `div`: `.intro`
- 2番目の `div`: `.c-blog-card` カード1
- 3番目の `div`: `.c-blog-card` カード2

この場合、`.c-blog-card:nth-of-type(-n + 2)` に当たるのはカード1だけ。  
カード2は `.c-blog-card` としては2番目でも、`div` としては3番目なので外れる。

## カード一覧での実務判断

```css
.blog-articles__list > .c-blog-card:nth-child(-n + 2) {
  grid-column: span 3;
}
```

これは、`.blog-articles__list` 直下の子要素全体の1〜2番目で、かつ `.c-blog-card` を持つ要素に当たる。

```html
<div class="blog-articles__list">
  <article class="c-blog-card">カード1</article>
  <article class="c-blog-card">カード2</article>
  <article class="c-blog-card">カード3</article>
</div>
```

- 直下にカードだけが並ぶなら、1枚目・2枚目のカード指定として自然。
- 見ているのはタグ種類ではなく、一覧内の並び順と配置。

## 明示クラスへ切り替える条件

```html
<article class="c-blog-card c-blog-card--large">カード1</article>
<article class="c-blog-card c-blog-card--large">カード2</article>
```

```css
.c-blog-card--large {
  grid-column: span 3;
}
```

次の場合は、順番セレクタより modifier の方が読みやすい。

- カード一覧の中に広告、見出し、空状態、バナーなどが混ざる可能性がある。
- 1枚目・2枚目という順番より「大きいカード」という意味を明示したい。
- CMSやJS生成で並び順が変わる可能性がある。
- 後から他人がHTML構造を変更する前提がある。

## 判断基準

- タグ種類ごとの順番を見たい: `:nth-of-type()`
- 親直下の並び順を見たい: `:nth-child()`
- クラスを持つ要素だけの意味差分を示したい: modifier
- カード一覧の配置差分: 直下にカードだけなら `:nth-child()`、混在可能性があるなら modifier
