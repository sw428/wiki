# 02_インラインと行の仕組み

## 目的

- インライン要素の高さ・基準線・ズレ方を説明できるようにする。

## ルール

- 行内レイアウトは「[行ボックス](../../03_雑記/知識辞書（試運用）/02_単語カード/行ボックス.md)」と「[ベースライン](../../03_雑記/知識辞書（試運用）/02_単語カード/ベースライン.md)」を基準に並ぶ。
- `line-height` は [行ボックス](../../03_雑記/知識辞書（試運用）/02_単語カード/行ボックス.md) の高さを決める中心パラメータ。
- `inline-block` は外側では行内要素として [行ボックス](../../03_雑記/知識辞書（試運用）/02_単語カード/行ボックス.md) に参加し、内側では `block` 的に `width` / `height` / `padding` を扱える。
- 文字自体は [テキストノード](../../03_雑記/知識辞書（試運用）/02_単語カード/テキストノード.md) で、指定反映は [CSS適用境界](../../03_雑記/知識辞書（試運用）/02_単語カード/CSS適用境界.md) を分けて確認する。

## 例

```css
p {
  font-size: 16px;
  line-height: 1.5; /* 行ボックスは 24px 相当 */
}

.badge {
  display: inline-block;
  padding: 2px 8px;
  background: #eef;
  vertical-align: middle;
}
```

- `vertical-align` は行内での基準合わせに効く。
- 画像や `inline-block` の下に隙間が見えるときは、[ベースライン](../../03_雑記/知識辞書（試運用）/02_単語カード/ベースライン.md) 整列が原因になりやすい。

## フォント中心と視覚中心のズレ

- フォントには `ascender` / `descender` / `baseline` のメトリクスが常にある。
- 文字が大文字だけでも、フォントメトリクス上の下側領域は残る。
- そのため「数学的中央」と「視覚的中央」は一致しないことがある。

実務では次の微調整が普通に発生する。

```css
.button__label {
  display: inline-flex;
  align-items: center;
  transform: translateY(1px);
}
```

## どの箱を基準に揃えているか

| 名称 | 正体 | 主に効く場所 |
|---|---|---|
| font box | フォント内部の仮想箱 | 文字の描画 |
| line box | 行全体の箱（`line-height`） | 行内配置 |
| border box | `content + padding + border` | レイアウト単位 |
| flex item box | flex配置対象の箱 | `align-items` など |

## `inline-block` と `inline-flex` の違い

- `inline-block`: 外側は行内文脈（[行ボックス](../../03_雑記/知識辞書（試運用）/02_単語カード/行ボックス.md) / [ベースライン](../../03_雑記/知識辞書（試運用）/02_単語カード/ベースライン.md)）に参加し、内側は `block` 的に扱う
- `inline-flex`: 外側は行内文脈に参加しつつ、内側の子要素配置は `align-items` など flex 規則で制御できるため、縦揃えを安定させやすい

補足:
- フォントメトリクスの影響が「ゼロ」になるわけではない
- 変わるのは主に「揃えの基準」

## 画像リンクで起きる基準線由来の下隙間

- `inline-flex` は「内側がflex」でも「外側はinline」のまま。
- そのため、画像リンクのような要素では baseline/descender 由来の下隙間が残ることがある。
- 画像枠を安定させたいときは、次を基準にする。

```css
.media-link {
  display: block;
}

.media-link > img {
  display: block;
}
```

- `display: flex` で隙間が消えるケースもあるが、画像1枚リンクなら `block` の方が意図が明確で副作用が少ない。

## テキストノードとタグ要素の分離

- HTML の文字そのものは [テキストノード](../../03_雑記/知識辞書（試運用）/02_単語カード/テキストノード.md) として存在する。
- `p` / `span` / `button` などは、その文字を包むタグ要素。
- CSS の指定先は基本的に要素であり、テキストノードへ直接指定するわけではない。
- ただし要素へ指定した継承プロパティや整列ルールが、結果としてテキスト描画に使われる。

```html
<p class="lead">Hello</p>
```

```css
.lead {
  text-align: center;
  color: red;
  font-size: 16px;
}
```

- CSS が当たる先は `.lead` 要素。
- `Hello` はテキストノード。
- `.lead` は `Hello` から見た親要素（テキストを直接包む要素）。
- `text-align: center` は、`.lead` の内容領域内でインライン内容を中央寄せするルールとして働く。

## `text-align` と中央寄せ責務の分離

`center` 系指定は「何を中央にするか」を分けて考える。

- `text-align: center;`  
  テキストノードを直接包む要素の内容領域内で、インライン内容（テキスト、inline要素）を中央寄せする。
- `margin-inline: auto;`  
  ブロック要素の箱そのものを親の中で中央に寄せる。
- `display: flex; justify-content: center;`  
  flex文脈で子要素を主軸中央に寄せる。

補足:
- `text-align` はテキストノードに直接指定するのではなく、テキストノードを含む要素の整列ルールとして効く。
- その結果として文字が中央に「見える」。

## 継承プロパティとテキスト描画

- テキスト描画は、要素に指定された継承プロパティが子孫へ渡ることで成立する。
- 代表例:
  - `font-size`
  - `font-family`
  - `font-weight`
  - `line-height`
  - `color`
  - `letter-spacing`
- フォントメトリクスはこの中の一部概念（基準線や字面寸法）として扱う。

## 初期認識メモ（確度）

- 確度: 高
  - `line-height` が行の見え方に強く効く。
  - `inline-block` は行内文脈を持ちながら `width`/`height` が扱える。
- 確度: 中
  - 「中身の有無で基準の見え方が変わる」という観察（主因はベースライン整列と行ボックス）。
- 確度: 要修正
  - 「テキストがあれば左上基準、なければ baseline 基準」と断定するより、常に行内フォーマット文脈での整列として扱うほうが再現性が高い。

## 判断基準

- 行内ズレを見たら次を確認する。
1. `line-height`
2. `vertical-align`
3. 置換要素（`img` など）かどうか
4. `display: inline` / `inline-block` の違い
5. 対象が「文章の一部」か「独立したUI部品」か
