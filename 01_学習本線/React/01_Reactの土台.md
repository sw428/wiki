# 01_Reactの土台

## 目的

- Reactが何を楽にする道具かを先に固定する。

## ルール

- Reactは「画面を部品で組む」ための仕組みとして捉える。
- HTML/CSS/JSの役割分離は崩さない。

## 要点

- UIをコンポーネント単位で分割する
- 状態が変わると再描画される
- 画面の最終形は「状態から計算された結果」として出る

## 例

```jsx
function Hello() {
  return <h1>Hello</h1>;
}
```

## 判断基準

1. これは独立した部品か
2. この部品は何を受け取るか（Props）
3. この部品は何を内部で持つか（State）
