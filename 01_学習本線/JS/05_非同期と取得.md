# 05_非同期と取得

## このファイルの役割

このファイルは、JavaScriptで「あとから届く値」をどう扱うかを整理する場所。
すぐに使える値ではなく、将来届く値をどう待って、どう取り出すかを見る。

ここでは主に次を扱う。

- 非同期
- Promise
- fetch
- async / await
- response.json()

## この範囲の芯

非同期は、
「今はまだ値がない」
という状態を扱う仕組み。

同期的な処理では、その場ですぐ値が返る。
非同期では、すぐには値が返らず、あとで届く。

だから大事なのは、次を分けて考えること。

- 今ある値なのか
- まだ完成していない値なのか
- 完成したあとにどう使うか

## 非同期とは何か

非同期は、時間がかかる処理を待つための仕組み。

たとえば次は、その場ですぐ終わらないことがある。

- 通信
- データ取得
- ファイル読み込み
- 画像読み込み

このとき、JSは
「完成前の値をそのまま返すのではなく、後で結果が届く仕組み」
を使う。

## Promise

Promise は、あとで値が入る箱のようなもの。

```js
const result = fetch("/data.json");

console.log(result);
```

この時点で `result` に入っているのは、すぐ使えるデータそのものではない。
まだ完成していない状態の値。

つまり Promise は、

- 今はまだ中身が使えない
- でも後で結果が入る予定

という箱。

## fetch

fetch は、データを取りに行くための関数。

```js
const result = fetch("/data.json");
```

これが返すのは、データそのものではなく Promise。

つまり、

- `fetch()` = 通信を始める
- 返るのは Promise
- すぐ中身は使えない

という流れ。

## async / await

`await` は、Promise の完成を待つために使う。

```js
const response = await fetch("/data.json");
```

ここでやっていることは次の流れ。

1. `fetch()` を実行する
2. Promise が完成するまで待つ
3. 完成した結果を取り出す

### async

`await` は `async` の中で使う。

```js
async function loadData() {
  const response = await fetch("/data.json");

  console.log(response);
}
```

見方:

- `async` = この関数の中では非同期処理を扱う
- `await` = Promise の完成を待つ

## response.json()

`fetch()` で返る `response` は、まだ最終的なJSONデータそのものではない。

```js
async function loadData() {
  const response = await fetch("/data.json");
  const data = await response.json();

  console.log(data);
}
```

ここでは次の段階になっている。

1. 通信結果を受け取る
2. その中身を JSON として読む
3. JavaScriptで使える形に変える

大事な見方:

- `fetch()` の返り値 = Promise
- `await fetch()` の返り値 = `response` オブジェクト
- `await response.json()` の返り値 = JSで使えるデータ

## JSONとのつながり

JSONは、通信や保存で使う文字列形式。
でも、JSの中で操作するときはオブジェクトとして使いたい。

だから、

- 通信では JSON
- 処理では オブジェクト

という役割分担になる。

`response.json()` は、
JSON文字列をJSで使える形に変える動き
として見るとわかりやすい。

## よくある流れ

```js
async function loadData() {
  const response = await fetch("/data.json");
  const data = await response.json();

  console.log(data);
}
```

何をしているか分けると、

1. `/data.json` を取りに行く
2. 通信完了を待つ
3. レスポンスを受け取る
4. JSONをJSで使える形にする
5. そのデータを使う

## 同期処理との違い

### 同期

```js
const x = 10;
console.log(x);
```

その場で値が使える。

### 非同期

```js
const response = fetch("/data.json");
console.log(response);
```

その場では完成前の箱しかない。

だから非同期では、
値をすぐ使おうとしないことが大事。

## 非同期をどう見るか

非同期で混乱しやすいのは、
「値が返っているように見えるのに、まだ使えない」こと。

でも整理すると、

- Promise = 未来の値
- await = 完成待ち
- response.json() = 使える形に変換

なので、やっていること自体はそこまで複雑ではない。

つまり、非同期も
「形がそろってから次に進む」
という見方でつながる。

- 非同期は「今はまだ値がない」状態を扱う
