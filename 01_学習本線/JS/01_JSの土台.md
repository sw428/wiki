# 01_JSの土台

## このファイルの役割

このファイルは、JavaScriptの文法表を作る場所ではなく、
JavaScriptという言語が世界をどう切り分けるかを身体化する場所。

ここで狙うのは「書けること」そのものより、
挙動を見た瞬間に意味づけできる頭の回路を作ること。

## この章で作る感覚

- 値は「ただのデータ」ではなく、計算と判断の最終到着点
- 型は「分類ラベル」ではなく、振る舞いの契約
- 演算は「記号の暗記」ではなく、値を値へ変える操作
- JSの曖昧さは「例外」ではなく、最初から埋め込まれた設計思想

## 著者の立場（なぜこの順で説明するか）

### 1. 先に抽象世界を見せる

コンピュータは意味を読まない。
扱っているのは、値と操作だけ。

だから最初に見るべきは文法より先に次の4つ。

- 数
- 文字
- 真偽
- 空（ない状態）

### 2. 先にJavaScriptのクセを隠さない

JavaScriptは、厳密で止める言語というより、
解釈して進める言語。

> [!NOTE]
> 「なんとなく動いた」を成功扱いにしない。
> ここで育てるのは、違和感を拾える読解力。

## 認知アプローチ（この章の読み方）

1. 現象を見る
コードを実行し、まず結果を受け取る。

2. 値として言い直す
「何が返ったか」を型込みで言語化する。

3. 変換の瞬間を探す
型が変わった地点、真偽へ押し込まれた地点を特定する。

4. 忖度を疑う
JSが「親切に埋めた解釈」を見つける。

5. 再現条件を固定する
同じ現象を別入力で再現し、偶然を消す。

## 値（Value）

値は、プログラムが直接扱える最小の実体。

- 数値
- 文字列
- 真偽値
- `null`
- `undefined`

変数は主役ではない。
主役はいつも値。

## 型（Type）

型は、値に許された振る舞いのルール。

- `number` は計算できる
- `string` は連結できる
- `boolean` は分岐に使える
- `object` は構造を持てる

JSは動的型付けなので、型は実行時に決まる。
自由度が高いぶん、書き手の観察責任も高い。

## Numberの思想

JSの数値は、基本的に `number` 1種類。
整数と小数を分離せず、64bit浮動小数点で扱う。

設計の優先順位は次。

- 完璧さより実用
- 厳密性より一貫した運用

だから次のような挙動が出る。

- `0.1 + 0.2 !== 0.3`

## 演算子は「値変換器」

演算子は記号ではなく、値を別の値へ変える装置。

- `+` `-` `*` `/` `%`
- `==` `===`
- `&&` `||`

ここで押さえる芯は1つ。

- 式は評価されて値になる

## 文字列と真偽値

### 文字列（String）

文字列は人間の言葉を扱う型。
数値と混ざると、同じ `+` でも意味が連結へ切り替わる。

### 真偽値（Boolean）

`true` / `false` は判断の最小単位。
複雑な状態も、分岐地点では最終的に真偽へ還元される。

## `null` と `undefined`

JSは「ない」を2種類で持つ。

- `null`: 意図して空にした
- `undefined`: まだ値が確定していない

きれいな設計というより、現実運用の履歴が残った設計。
ここを曖昧にすると、後で状態管理が崩れる。

## 型変換（Type Coercion）

JSは必要に応じて黙って型を変える。

- 数値 + 文字列 -> 文字列
- 比較時に型を揃える
- 条件式で真偽値に変換する

これは便利機能でもあり、事故ポイントでもある。

## 比較（`==` と `===`）

### `==`（ゆるい比較）

- 型が違っても比較する
- 内部で型変換が起こる

### `===`（厳密比較）

- 型と値の両方が一致したときだけ真
- 暗黙変換を行わない

仕組み理解が浅い段階では `===` を基準にする。

## truthy / falsy

条件文に渡された値は、強制的に真偽へ変換される。

主な falsy:

- `false`
- `0`
- `""`
- `null`
- `undefined`
- `NaN`

それ以外は基本 truthy。
JSでは「厳密な論理」より「存在しているか」が分岐の実態になりやすい。

## 論理演算子は真偽値以外も返す

`&&` と `||` は、真偽値を返す専用機ではない。
値を返す。

- `A && B`: `A` が偽なら `A`、真なら `B`
- `A || B`: `A` が真なら `A`、偽なら `B`

この性質が、デフォルト値選択や安全な分岐に使われる。

## NaN と Infinity

### NaN

`NaN` は `number` 型のまま、計算不能を示す値。

- `NaN !== NaN`

### Infinity / -Infinity

範囲外の結果をエラーで止めず、値として保持する。

ここにもJSの設計姿勢が出る。

- 失敗を例外で断つより、値として流す

## 観察ドリル（この章の使い方）

```js
console.log(1 + "2");
console.log("5" == 5, "5" === 5);
console.log(Boolean(""), Boolean("0"));
console.log(null == undefined, null === undefined);
console.log(NaN === NaN, Number.isNaN(NaN));
```

このドリルで見る点:

- どこで型が変わったか
- どこで比較ルールが切り替わったか
- どこでJSが失敗を値として持ち続けたか

## この範囲の核心まとめ

- JSの最小単位は値
- 型は振る舞い契約
- 演算は値変換
- 式は評価されて値になる
- 暗黙の型変換は通常運転
- JSは「失敗を消す言語」ではなく「失敗を値として運ぶ言語」
