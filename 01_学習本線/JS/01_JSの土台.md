# 01_JSの土台

## このファイルの役割

このファイルは、JavaScriptの土台をまとめる場所。
文法を暗記する前に、JSが何をどう扱う言語なのかを整理する。

ここでは主に次を扱う。

- 値
- 型
- 演算子
- 式
- 文
- 変数
- 型変換
- 比較
- 真偽値

## JSは何を扱う言語か

JavaScriptは、まず「値」を扱う言語。
コンピュータは意味を理解するのではなく、値と操作を扱う。

人間が考える次のものを、機械が扱える形に落とすのがプログラミング。

- 数
- 文字
- 真偽
- 空の状態

この意味で、JSは
「文法を覚える前に、値をどう扱うかを見る言語」
として理解したほうが入りやすい。

## 値と型

### 値

値は、プログラムが直接扱う最小単位。

例:

- 数値
- 文字列
- 真偽値
- `null`
- `undefined`

変数より先に値がある。
計算も判断も、最後は値として扱われる。

### 型

型は、値の振る舞いルール。

- `number` は計算できる
- `string` はつなげられる
- `boolean` は条件判断に使う
- `object` は構造を持てる

大事なのは、型は見た目ではなく、
「何ができて、何ができないか」で決まること。

JSは動的型付けなので、実行時に型が決まる。
自由だが、そのぶん書いた側が型を意識しないと事故る。

## 演算子

演算子は、値を別の値に変えるためのもの。

例:

- `+`
- `-`
- `*`
- `/`
- `%`
- `===`
- `&&`
- `||`

演算子は文法記号というより、
「値 -> 値」を作る装置と見たほうがわかりやすい。

特に重要なのは、JSでは
「式の結果は必ず値になる」
ということ。

## 式と文

### 式

式は、評価されて値になるもの。

例:

- `3 + 4`
- `Math.max(5, 10)`

式は他の式の中に入れ子で書ける。
計算の単位。

### 文

文は、実行される単位。

例:

- `let x = 10;`
- `console.log(x);`

プログラムは文の並びでできている。
文は式を含むことがあるが、文自体は
値を返すことより、効果を起こすことが目的。

つまり、

- 式 = 値を作る
- 文 = 実行して効果を出す

という切り分け。

## 変数

JSでは、変数は値を扱うための名前。

`let score = 10;`

ただし、変数を「箱」と思いすぎるとズレやすい。
ここでは

「変数 = 値を指す札」

と見たほうが合う。

- 変数は実体ではない
- 実体は値
- 変数はその値への入口

この感覚は、あとでプリミティブとオブジェクトの違いを理解するときに大事。

### 変数名の基本

- 先頭を数字にできない
- `$` と `_` は使える
- 予約語は使えない
- 意味が伝わる名前にする

## 型変換

JSは、勝手に型を変えることがある。
これが型変換。

例:

- 数値 + 文字列 -> 文字列
- 条件式の中 -> 真偽値に寄せる
- 比較時 -> 同じ型に寄せようとすることがある

JSは
「エラーで止めるより、とりあえず動かそうとする」
傾向がある。

この「親切さ」が便利なこともあるが、同時に地雷にもなる。

## 比較

### `==`

ゆるい比較。
型が違っても、内部で型変換して比較することがある。

### `===`

厳密比較。
型も値も一致している必要がある。

基本は `===` を優先したほうが安全。
`==` は仕組みを理解していないと事故りやすい。

## 真偽値と存在判定

JSでは、条件文に来た値は `true` / `false` に変換される。

### falsy

代表例:

- `false`
- `0`
- `""`
- `null`
- `undefined`
- `NaN`

それ以外は基本的に truthy。

ここで大事なのは、JSでは
厳密な論理だけでなく「値があるかどうか」がかなり重要な判断軸になること。

## 論理演算子

JSの `&&` と `||` は、ただの `true` / `false` を返すものではない。

### `A && B`

- `A` が偽なら `A`
- `A` が真なら `B`

### `A || B`

- `A` が真なら `A`
- `A` が偽なら `B`

つまり、論理演算子は
真偽判定だけでなく、値を返す制御の道具。

この性質は、デフォルト値や存在確認でよく使う。

## NaN と Infinity

### NaN

`NaN` は `number` 型だが、計算不能を表す。

- 数値型ではある
- でも普通の数ではない
- `NaN !== NaN`

JSはここでも、エラーで止めずに失敗を値として持ち続ける。

### Infinity

`Infinity` / `-Infinity` は、数値の範囲外を表す値。

これも例外ではなく `number` の一種として扱う。
JSは「壊れたら止まる」より、
「壊れても値として流し続ける」側の言語。

## この範囲の見方

この範囲のコードは、正解を暗記するためというより、次を見るためのもの。

- どこで型が変わったか
- どこで比較が起きたか
- どこで真偽値に押し込まれたか

つまり、ここで育てたいのは
「なんとなく動いた」を信用しない目。

## この範囲のまとめ

- JSの最小単位は値
- 型は値の振る舞いルール
- 演算子は値を別の値に変える
- 式は値を作る
- 文は実行して効果を出す
- 変数は実体ではなく、値への入口
- JSは勝手に型を変えることがある
- 比較や条件では「存在判定」がかなり重要
- JSはエラーで止めるより、値として流し続ける傾向がある
