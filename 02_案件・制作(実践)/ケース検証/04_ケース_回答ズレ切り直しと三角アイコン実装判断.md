# 04_ケース_回答ズレ切り直しと三角アイコン実装判断

## 1. 事実（案件文脈）

- 対象は、リアルタイム対話ログの混在回。
- 同一ログ内で次の論点が混在した。
  - 質問意図と無関係な回答が返る（照準ずれ）
  - VS Code操作（`F4` / `F8` での移動）
  - 三角形アイコン実装（CSS）
  - `::after` とHTML要素（`span`）の使い分け
  - `SVG` / `PNG` の選択とサイズ設計
- フロントエンド実務と無関係な個人話題は記録対象外にした。

## 2. 判断プロセス

### ケース1: 質問と回答の照準ずれをどう止めるか

#### 候補A: そのまま同じスレッドで質問を重ねる

- 判定: 不採用
- 理由: 混在トピックが残ると、さらに別話題へ逸れやすい。

#### 候補B: 質問対象を1行で再固定し、出力形式を先に指定する

- 判定: 採用
- 理由: 回答が「何に答えるか」を先に拘束できる。
- 例:
  - `今は三角形アイコンだけ。CSS例 + 使い分け3行で。`
  - `今はSVGとPNGの判断だけ。用途/サイズ/書き出し比率で。`

### ケース2: 三角形アイコンをどう実装するか

#### 候補A: すべてHTML要素（`span`）で持つ

- 判定: 条件付き採用
- 理由: JSで状態制御する部品なら、DOM要素として持つ方が扱いやすい。

#### 候補B: 装飾用途は `::before` / `::after` で作る

- 判定: 採用（既定）
- 理由: 意味を持たない矢印なら、HTMLを汚さずに管理できる。

```css
.button {
  display: inline-flex;
  align-items: center;
  gap: 8px;
}

.button::after {
  content: "";
  width: 0;
  height: 0;
  border-top: 4px solid transparent;
  border-bottom: 4px solid transparent;
  border-left: 6px solid currentColor;
  flex-shrink: 0;
}
```

#### 候補C: 三角も常にSVG化する

- 判定: 条件付き採用
- 理由: 複雑形状・正確な制御・アニメーション要件がある時はSVGが有利。単純矢印ならCSSで十分。

### ケース3: `SVG` と `PNG` の選択

#### 候補A: アイコン/線画はSVG優先

- 判定: 採用（既定）
- 理由: 拡大縮小で劣化しにくく、色変更・CSS連携もしやすい。

#### 候補B: 写真/複雑テクスチャはPNG

- 判定: 条件付き採用
- 理由: ラスター画像向き。透過が必要な静止素材には有効。
- 補足: 写真系は配信効率で `WebP` / `AVIF` を優先検討し、PNG常用は避ける。

#### サイズの決め方（実務メモ）

- SVG:
  - 元データは `viewBox` を正しく定義し、表示サイズはCSSで指定する。
  - 例: アイコン表示 `16px / 20px / 24px` を基準化。
- PNG:
  - 表示寸法を先に固定し、書き出しは `1x` と必要に応じて `2x` を用意する。
  - HTMLでは表示寸法に合わせて `width` / `height` を明示する。

### ケース4: `F4` / `F8` を連打してよいか

#### 候補A: `F4` と `F8` を混在で連打する

- 判定: 非推奨
- 理由: キーバインド文脈が揺れると、想定外の場所へ移動しやすい。

#### 候補B: エラー巡回は `F8`（次）/ `Shift+F8`（前）で統一する

- 判定: 採用
- 理由: VS Codeの問題巡回として意図が明確で、再現性が高い。
- 補足:
  - 連打自体は問題ないが、修正後は `Problems` 件数が更新されたかを都度確認する。
  - `Ctrl+G` の行番号ジャンプを併用すると、往復確認が速い。

## 3. 案件固有として残すもの（02側）

- 「照準ずれ」が発生した回は、混在トピックをいったん分離してから再質問する運用が有効だった。
- 本ケースでは「矢印装飾は疑似要素」が中心だったが、実装対象のUI仕様次第でDOM要素管理へ切り替える余地がある。

## 4. 本線へ戻すもの（01/04側）

- 再利用可能な原理:
  - 装飾は疑似要素、状態を持つ部品は実要素で管理する。
  - 画像形式は「可変倍率が必要か」「写真か図形か」で先に分岐する。
  - エラー巡回の操作系はショートカットを混在させず、1系統で固定する。
  - 回答ずれ時は、質問対象1行固定 + 出力形式指定で切り直す。
- 判断順:
  1. その三角形は意味を持つか（持たないなら疑似要素）
  2. 素材は図形か写真か（図形ならSVG、写真/質感ならラスター）
  3. 巡回目的は何か（エラー巡回は `F8` 系で統一）
  4. 回答がずれたら、対象を1トピックに再固定
- 例外条件:
  - JSで回転/開閉状態を直接制御する必要がある場合は、`span` 要素を置く。
  - 細かなピクセル表現が必要なUI素材はPNG運用を許容する。
