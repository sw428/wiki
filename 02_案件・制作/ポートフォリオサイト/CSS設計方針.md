## レベル2の到達目標

- BEMで意味と見た目を分離できる
- レイアウト責務を`l-`へ分離できる
- 余白設計をルール化して崩れにくくできる
- 他人が読んで修正しやすいCSSを維持できる

## CSS設計方針（BEM × レイアウト分離）

### 基本思想

- CSSはクラスベース・低詳細度で記述する
- HTMLタグの意味と、見た目・余白・配置の責務を分離する
- 将来の仕様変更・他者修正を前提にする

### 命名

- Block: `.about`, `.card`, `.header`
- Element: `.about__title`, `.card__img`
- Modifier: `.card--featured`, `.card__img--wide`

ルール:

- `__`は1段まで
- Modifierは状態・バリエーションのみ
- レイアウト都合をBlockへ直接持たせない

### レイアウト分離

- 並び・余白・配置は`l-`クラスで管理する
- 名前は見た目ではなく役割で付ける

```css
.l-list {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}
```

### 余白ルール

- Block間隔は上側Blockの`margin-bottom`で管理
- `margin-top`は例外調整のみ
- `margin`/`padding`のショートハンドは使わない

### 文章（p）の扱い

- `p`自体に余白を持たせない
- 余白は親で制御する

```css
p {
  margin: 0;
}

.richtext > * + * {
  margin-top: 1em;
}
```

### 画像の初期実装

- 初期は`img/dummy.jpg`で組む
- 比率崩れ防止に`aspect-ratio`と`object-fit`を先に固定

```css
.hero__img {
  width: 100%;
  aspect-ratio: 16 / 9;
  object-fit: cover;
  background-color: #eee;
}
```

## 制作順序（実務編レベル2）

1. セクション構造を確定する（ヘッダー/メイン/フッター、各Block）
2. BlockとElement名を決める（意味ベース）
3. レイアウト責務を`l-`へ分離する（並び・配置・gap）
4. 余白ルールを先に固定する（`margin-bottom`中心）
5. ダミー画像とテキストで骨組みを先に完成させる
6. 見た目（色・線・タイポ）をBlock側へ実装する
7. 状態差分をModifierで追加する
8. 影響範囲を確認しながら調整する（詳細度競争を避ける）

## 判断基準（迷ったら）

- 再利用可能性がある: 分離する
- 他者修正が想定される: 分離する
- CMS / Markdownが絡む: `p`を汚さない
- 1ページ短期案件: 最小限の割り切りは許容

## 完了チェック

- Blockとレイアウトが混ざっていない
- `p`に直書き余白を入れていない
- `margin-top`濫用がない
- 3回以上出るレイアウトを共通化できている
- クラス名だけで意図が読める
