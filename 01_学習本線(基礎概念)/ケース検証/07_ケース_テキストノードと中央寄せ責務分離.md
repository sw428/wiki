# 07_ケース_テキストノードと中央寄せ責務分離

## このページの扱い

- 単一議題（DOMノード分類と `text-align` 解釈）

## ケース1_テキストノードは子要素か

### 現象

- `p` の中の文字を「子要素」と呼んでよいかで混線した。

### 前提条件

- 例: `<p>こんにちは</p>`
- DOM上では `p` 要素の内側にテキストがある。

### 仮説

- 文字はDOMノードではあるが、要素ノードではない可能性が高い。

### 検証ログ（1回に1条件）

1. DOMノード種別を分けて確認  
   出力: `要素ノード` と `テキストノード` は別種  
   判定: 文字は「子ノード」だが「子要素」ではない。

### 採用した形

- 採用文:
  - テキストノードは `p` 要素の**子ノード**。
  - ただし**子要素ではない**。
- 採用理由（条件 / 理由 / 例外）:
  - DOMの精密な説明・JS操作の説明で用語を揃えるため。
  - 会話上の略語として「中の文字」と言うのは可だが、技術文では分離する。

### 見送った形

- `テキストはpの子要素`  
  - 不採用理由: Node種別の区別が崩れ、`children` / `childNodes` の理解を誤りやすい。

## ケース2_`children` と `childNodes` の違い

### 現象

- DOM操作時に「どちらを使うか」で対象がズレる。

### 前提条件

- 例:
  - `<p id="t">こんにちは<span>世界</span></p>`

### 仮説

- `children` は要素のみ、`childNodes` はテキストノードを含む。

### 検証ログ（1回に1条件）

1. 2APIの戻りを比較  
   出力: `children -> [span]`, `childNodes -> [text, span]`  
   判定: 仮説通り。

### 採用した形

- 要素だけを走査するなら `children`。
- 文字ノードも含めて扱うなら `childNodes`。

## ケース3_`text-align: center` は何を中央にしているか

### 現象

- `text-align` と `margin-inline: auto` と `flex` 中央寄せが混線しやすい。

### 前提条件

- `text-align` は要素に指定するプロパティ。

### 仮説

- `text-align` は「箱そのもの」ではなく、箱内インライン内容の整列に効く。

### 検証ログ（1回に1条件）

1. `text-align: center` を親に適用  
   出力: 文字やインライン内容が中央寄せ  
   判定: 箱自体は移動しない。
2. `margin-inline: auto` をブロック要素へ適用  
   出力: 要素の箱自体が中央に寄る  
   判定: `text-align` と責務が別。

### 採用した形

- `text-align`: インライン内容の整列
- `margin-inline: auto`: 箱の中央寄せ
- `justify-content` / `align-items`: flex文脈での子要素整列

## ケース4_「フォントメトリクスのみ継承」の表現

### 現象

- 継承説明が「フォントメトリクスのみ」に寄りすぎると誤解が出る。

### 前提条件

- CSSは要素へ指定され、継承プロパティが子孫へ渡る。

### 仮説

- 継承はフォントメトリクス限定ではなく、テキスト系継承プロパティ全体で説明すべき。

### 検証ログ（1回に1条件）

1. 継承対象を整理  
   出力: `font-size` / `font-family` / `font-weight` / `line-height` / `color` / `letter-spacing` など  
   判定: フォントメトリクスはその一部概念として扱うのが適切。

### 採用した形

- 「継承されるテキスト系プロパティが、結果としてテキストノード描画に使われる」と表現する。
