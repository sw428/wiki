# 05_ジェネリクス基礎

## 目的

- 型を固定せず、再利用可能な型設計の最小形を理解する。

## ルール

- まず具体型で動かしてからジェネリクス化する
- 汎用化しすぎない

## 要点

- ジェネリクスは「型引数」を受け取る仕組み
- 同じロジックを複数型で安全に使い回せる

## 例

```ts
function firstItem<T>(items: T[]): T | undefined {
  return items[0];
}

const a = firstItem<number>([10, 20, 30]);
const b = firstItem<string>(["A", "B", "C"]);
```

## 判断基準

1. 同型ロジックの重複があるか
2. `T` で一般化する価値があるか
3. 戻り値が安全に表現されているか
