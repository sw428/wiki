# 04_イベントとフォーム

## 目的

- Reactでのイベント処理とフォーム制御を最低限の型で使えるようにする。

## ルール

- 入力値はStateで持つ
- 送信時は `preventDefault()` を使う

## 要点

- `onClick` / `onChange` / `onSubmit` を基本にする
- フォーム入力は制御コンポーネントで扱う

## 例

```jsx
import { useState } from "react";

function SearchForm() {
  const [keyword, setKeyword] = useState("");

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(keyword);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={keyword}
        onChange={(e) => setKeyword(e.target.value)}
      />
      <button type="submit">Search</button>
    </form>
  );
}
```

## 判断基準

1. 入力の単一情報源がStateになっているか
2. 送信時の副作用が分離されているか
3. UI状態とデータ状態を混同していないか
