# 08_基本文法メモ

## このファイルの役割

このファイルは、JavaScriptを書くときの最小ルールをまとめる場所。
文法エラーや書式ブレで止まる回数を減らすための「手元チェック」として使う。

## JS全体の書き方ルール

### 1. 大文字と小文字は別物

```js
const name = "Taro";
console.log(name); // OK
console.log(Name); // ReferenceError
```

- `name` と `Name` は別の識別子。

### 2. 文字列は引用符で囲む

```js
const msg1 = 'hello';
const msg2 = "hello";
```

- `'` か `"` のどちらでも動く。
- プロジェクトのルールがなければ、どちらかに統一する。
- HTML属性で `"` が多い現場では、JS側を `'` に寄せると見分けやすい。

### 3. 文末のセミコロン

```js
const a = 1;
const b = 2;
console.log(a + b);
```

- ASI（自動セミコロン挿入）で動くことはあるが、誤判定を避けるため付ける方針で統一する。

### 4. 半角英数字で書く

- 全角スペースや全角記号が混ざるとエラー原因になりやすい。
- 記号まわりで違和感が出たら、まず全角混入を疑う。

## 変数名・定数名ルール

### 使える文字

- 英字
- 数字（先頭は不可）
- `_`
- `$`

### 使えないもの

- スペース
- 予約語（`if`, `const`, `class` など）

### 実務での命名目安

```js
const userName = 'Taro';
const MAX_RETRY = 3;
```

- 変数: `camelCase`
- 定数: `UPPER_SNAKE_CASE`（チームルールに従う）

## テンプレートリテラル

テンプレートリテラルは、バッククォート `` ` `` で囲む文字列。

```js
const name = 'Taro';
const total = 1200;

const message = `こんにちは、${name}さん。合計は${total}円です。`;
```

### できること

- `${}` で変数や式を埋め込める
- 複数行をそのまま書ける

```js
const multi = `1行目
2行目`;
```

## 条件分岐（if / else if / else）

```js
const score = 85;

if (score >= 90) {
  console.log('とても良い');
} else if (score >= 70) {
  console.log('まあまあ良い');
} else {
  console.log('がんばろう');
}
```

読み方:

1. 最初に `if` を判定
2. 偽なら `else if` を上から順に判定
3. どれも偽なら `else`
