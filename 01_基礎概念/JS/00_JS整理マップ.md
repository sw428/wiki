# 00_JS整理マップ

## このファイルの役割

このファイルは、JavaScriptメモ全体の入口。
各ファイルに何を書いたかと、どの順で読めばいいかをまとめる。

## このフォルダの見方

JavaScriptは、全部を一気に覚えるより、次の順で分けて見ると追いやすい。

- 土台
- データ
- DOM
- UI
- 非同期
- クロージャ
- 実行環境とツール
- 関数設計
- 基本文法
- モジュール分割
- 補助知識

また、JavaScriptは使われる場所によって重要になる型・パターンが変わる。

- Web制作: DOM操作、イベント、class / 属性の切り替え、フォーム、アクセシビリティ状態
- フロントエンドアプリ: 状態管理、コンポーネント、API通信、非同期処理
- Node.js: サーバー処理、ファイル操作、API、DB接続
- WordPress制作: メニュー開閉、スライダー、フォーム補助、Ajax、テーマ内UI操作

最初からJavaScript全体を均等に覚えるより、自分が使う分野で繰り返し出る処理パターンを先に掴む。

## 認知アプローチ（知識に至る手順）

「文法暗記」ではなく、次の手順で判断力を作る。

1. 目的に戻る
   まず「今何を作りたいか（例: ポートフォリオUI）」を固定する。
2. データの形を確認する
   値は何型か、文字列かオブジェクトか、配列かを先に判定する。
3. 実行場所を確認する
   ブラウザかNode.jsかを切り分け、使えるAPI前提を揃える。
4. 失敗点を最小化する
   1行単位で `console.log` し、期待値と実値の差だけを見る。
5. 変更を局所化する
   DOM取得・イベント・状態更新を分離し、どの層のミスか特定する。
6. 学習ログを残す
   正解だけでなく「誤解した点」「修正理由」を短く残す。
7. 知識と再現を分ける
   「知っている」ではなく「条件付きで再現できる」を到達基準にする。

## 重複を活かす読み方

同じ話題が別ファイルに出るのは、整理不足だけが理由ではない。
このフォルダでは、あえて角度を変えて重複させることがある。

- `01_JSの土台` は「言語の世界観」を掴むための重複
- `04_データと配列・オブジェクト` は「構造として扱う」ための重複
- `02_DOMとイベント` は「画面実装で再現する」ための重複

重複を見つけたら削る前に、次を確認する。

1. 何を理解させるための重複か（概念 / 構造 / 実装）
2. どの判断ミスを減らすための重複か
3. どの場面で再利用する前提か

## 没入モード（読むときの視点）

1. 世界観モード
   「JSはなぜこう振る舞うか」を読む。

2. 実験モード
   短いコードを実行し、結果のズレを観察する。

3. 制作モード
   ポートフォリオ実装で使うなら、どの判断に効くかを紐づける。

## このフォルダの芯

このフォルダ全体の芯はこれ。

- JSは値を扱う
- JSはデータの形を読む
- DOMもオブジェクトとして読む
- UI実装はイベントと状態変更で作る
- Web制作のJSは、HTML / CSS / ARIA の間で状態を橋渡しする
- 何を受け取り、どの型に変換し、どこへ返すかを見る
- 非同期は未来の値を扱う
- クロージャは参照の鎖で理解する
- 関数設計とモジュール分割で実装の責務を分ける
- 補助知識は必要時に切り出して使う
- 実行環境とツールは役割を分けて見る

## 制作で使う最小セット（8割）

ポートフォリオ制作で先に押さえる実用チェック。
「わからない単語が出ても、この順に戻る」を前提にする。

### 1. 土台（型と構造）

- [ ] 型を見分けられる（`string` / `number` / `boolean` / `object` / `array` / `null` / `undefined`）
- [ ] オブジェクトと配列の違いを説明できる
- [ ] JSON文字列とオブジェクトの違いを説明できる

参照:

- [01_JSの土台](./01_JSの土台.md)
- [04\_データと配列・オブジェクト](./04_データと配列・オブジェクト.md)

### 2. DOM操作

- [ ] `querySelector` / `querySelectorAll` で要素取得できる
- [ ] `textContent` / `classList` / `value` を使える
- [ ] `getAttribute`（初期値）と `プロパティ`（現在値）の違いを使い分けられる
- [ ] HTML属性を文字列として読み、JS側で真偽値や数値に変換して戻せる

参照:

- [02_DOMとイベント](./02_DOMとイベント.md)

### 3. イベントとUI

- [ ] `addEventListener` の第2引数に関数を渡せる
- [ ] `if` で状態分岐し、`classList.toggle` で開閉を作れる
- [ ] 現在の状態から次の状態を決め、class / aria / hidden へそろえて反映できる
- [ ] フォームで `preventDefault` と `value` を使える

参照:

- [02_DOMとイベント](./02_DOMとイベント.md)
- [03_UI実装パターン](./03_UI実装パターン.md)

- [ ] 大文字小文字ミスを潰せる
- [ ] 文字列クォート・セミコロン方針を揃えられる
