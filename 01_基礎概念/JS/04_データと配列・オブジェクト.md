# 04_データと配列・オブジェクト

## このファイルの役割

このファイルは、JavaScriptで扱う「データの形」をまとめる場所。
JSを文法ではなく、データの形を読む言語として理解するための土台にする。

ここでは主に次を扱う。

- オブジェクト
- プロパティ
- 配列
- JSON
- プリミティブ
- 参照とコピー

## この範囲の芯

JSは暗記する言語というより、
データの形を読んで、形を変換していく言語として見ると整理しやすい。

大事なのは、次を見分けること。

- 今それが何の型か
- 構造を持っているか
- 取り出せるか
- 変換できるか

## オブジェクト

オブジェクトは、キーと値の組を持つデータ。

```js
const user = {
  name: "Taro",
  age: 20
};
```

ここでは

- `name`
- `age`

がプロパティ名で、

- `"Taro"`
- `20`

が値。

オブジェクトは
「ラベル付きの箱」
として見るとわかりやすい。

## プロパティ

オブジェクトの中の値は、プロパティを使って取り出す。

## 見た目が似ていても別扱い

次の3つは似て見えても別物。

```js
name;      // 変数を読む
obj.name;  // プロパティを読む
run();     // 関数を実行する
```

- `name`: 変数
- `obj.name`: オブジェクトの中のプロパティ
- `run()`: 関数呼び出し

同じ `name` という文字が出てきても、種類が違えば衝突しない。

```js
const name = "outer";
const obj = { name: "Taro" };
```

これは「スコープが違うから」以前に、
そもそも `変数名` と `プロパティ名` で種類が違うため同時に成立する。

### ドット記法

```js
const user = { name: "Taro" };
console.log(user.name);
```

固定の名前でアクセスするときに使う。

### ブラケット記法

```js
const user = { name: "Taro" };
const key = "name";

console.log(user[key]);
console.log(user["name"]);
```

文字列や変数を使ってプロパティ名を指定したいときに使う。

### 違い

- `user.name` は固定名
- `user[key]` は動的

つまり、

- ドット記法 = もう名前が決まっている
- ブラケット記法 = 後から名前を渡す

という違い。

## 文字列とオブジェクトの違い

文字列は、ただの文字の並び。
構造としてのキーを持たない。

```js
const text = '{ "name": "AAA" }';
console.log(text.name); // undefined
```

一方、オブジェクトは構造を持つ。

```js
const obj = { name: "AAA" };
console.log(obj.name); // "AAA"
```

ここで大事なのは、

- 文字列 = 読めるだけ
- オブジェクト = 中身を取り出せる

という違い。

## `:` が出たときの見分け

### オブジェクトのプロパティ定義

```js
const obj = {
  name: "Taro"
};
```

この `name: "Taro"` はプロパティ定義。

### ラベル文（別文法）

```js
function test() {
  name: "Taro";
}
```

これはオブジェクトではなくラベル文として解釈される。
同じ `:` でも文脈で意味が変わる。

## JSON

JSON は、オブジェクトっぽく見える文字列。

```js
const json = '{ "name": "AAA" }';
```

これは見た目はオブジェクトっぽいが、まだ文字列。
そのままではプロパティにアクセスできない。

### JSON.parse

文字列をオブジェクトに変える。

```js
const json = '{ "name": "AAA" }';
const obj = JSON.parse(json);

console.log(obj.name);
```

### JSON.stringify

オブジェクトを文字列に変える。

```js
const obj = { name: "AAA" };
const json = JSON.stringify(obj);
```

### 役割分担

- 通信・保存 = JSON文字列
- 処理・操作 = オブジェクト

## 配列

配列は、順番つきのデータの集まり。

```js
const nums = [1, 2, 3];
```

オブジェクトが「名前付きの箱」なら、配列は「順番つきの箱の列」。

### 配列で大事なこと

- 順番がある
- 同じ処理を繰り返しかけやすい
- 1個ずつ見て処理しやすい

## 配列操作

### map

1個ずつ変換して、新しい配列を作る。

```js
const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2);
```

- 入力 = 配列
- 処理 = 1個ずつ変換
- 出力 = 新しい配列

### filter

条件に合うものだけ残して、新しい配列を作る。

```js
const nums = [1, 2, 3];
const result = nums.filter(n => n >= 2);
```

### reduce

配列全体を1つの値にまとめる。

```js
const nums = [1, 2, 3];
const sum = nums.reduce((acc, n) => acc + n, 0);
```

### 見方

- `map` = 配列 -> 配列
- `filter` = 配列 -> 配列
- `reduce` = 配列 -> 1つの値

つまり、全部データ変換。

## プリミティブとオブジェクト

ここはかなり重要。

### プリミティブ

例:

- `string`
- `number`
- `boolean`
- `null`
- `undefined`

プリミティブは、値そのものを扱う。

### オブジェクト

例:

- `{}`
- `[]`
- 関数

オブジェクトは、構造を持つ。

## 関数はオブジェクト

関数は「実行できるオブジェクト」。

```js
function test() {
  console.log("hello");
}

test.count = 0;
test.label = "sample";
```

ここで増えたのは「別のtest」ではなく、
同じ `test` 関数オブジェクトにプロパティを追加しているだけ。

## 関数呼び出しと関数プロパティは別

```js
function test() {
  console.log("hello");
}

test.count = 0;
console.log(test.count); // 0
test();
console.log(test.count); // 0
```

`test()` しただけでは `test.count` は自動で変わらない。
変えるなら、関数内で明示的に使う必要がある。

```js
function test() {
  test.count++;
}

test.count = 0;
test();
console.log(test.count); // 1
```

## `this` は関数自身ではない

`this` は「どの関数か」ではなく「どう呼ばれたか」で決まる。

```js
function test() {
  console.log(this);
}

test(); // this は test 自身とは限らない
```

```js
const obj = {
  run() {
    console.log(this);
  }
};

obj.run(); // this === obj
```

つまり、

- `test.count`: 関数オブジェクトのプロパティ
- `this.count`: 呼び出し元オブジェクト依存

で別物。

## 代入の違い

### プリミティブ

```js
let a = "hello";
let b = a;

b = "world";

console.log(a); // "hello"
console.log(b); // "world"
```

これは値のコピー。

### オブジェクト

```js
let a = { value: 10 };
let b = a;

b.value = 99;

console.log(a.value); // 99
```

これは実体のコピーではなく、参照の共有。

つまり、

- プリミティブ = 値をコピー
- オブジェクト = 参照をコピー

## 参照のイメージ

変数は実体そのものではなく、値への入口。

オブジェクトのときは、同じ実体を複数の変数が指せる。

```js
let a = { value: 10 };
let b = a;
```

この状態では、

- `a`
- `b`

が同じオブジェクトを見ている。

だから

```js
b.value = 99;
```

をすると、`a` から見ても変わる。

## 文字列が共有されない理由

```js
let a = "hi";
let b = a;

b = "hello";
```

これは、`b` が別の値を指し直しただけ。
`a` には影響しない。

さらに文字列は不変なので、

```js
let text = "hi";
text[0] = "H";
```

のように直接書き換えることはできない。

## DOMも同じ見方で読める

DOM操作も本質的にはオブジェクト操作。

```js
const p = document.querySelector("p");
p.textContent = "hello";
```

ここでやっているのは、

1. `p` というオブジェクトを取る
2. `textContent` というプロパティを書き換える

ということ。

つまり、画面を直接触っているというより、
オブジェクトを書き換えていると見たほうが整理しやすい。

## この範囲のまとめ

- オブジェクトはラベル付きの箱
- 配列は順番つきの箱の列
- JSONはオブジェクトではなく文字列
- `JSON.parse` で文字列をオブジェクトに変える
- ドット記法は固定名、ブラケット記法は動的指定
