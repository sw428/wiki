# 01_HTMLの土台

## 目的

- `display` やボックスモデルへ入る前に、HTMLが何としてブラウザに読まれるかを整理する。
- 「タグ」「要素」「属性」「テキストノード」「DOM」を混同せずに読める状態にする。

## まず持つ地図

HTMLは、見た目を直接作る文字列ではなく、ブラウザに解析されてDOMツリーになる元データ。

```txt
書いたHTML
-> ブラウザが解析する
-> 必要に応じて補正される
-> DOMツリーになる
-> JSやCSSの処理対象になる
```

この時点で、HTMLに書いた文字はそのまま一枚の文章として扱われるのではなく、要素ノードやテキストノードに分かれる。

## HTMLは「要素 + テキストノード」のツリー

HTMLドキュメントは、HTML要素とテキストノードを含むノードのツリーとして扱われる。

たとえば次のHTML。

```html
<p>こんにちは <strong>重要</strong></p>
```

DOM上では、ざっくり次のように見る。

```txt
p要素ノード
├─ テキストノード「こんにちは」
└─ strong要素ノード
   └─ テキストノード「重要」
```

つまり、テキストもDOM上では「何もない文字」ではなく、テキストノードとして扱われる。

## タグと要素は同じではない

タグは、HTML内で要素を作るための印。

```html
<p>こんにちは</p>
```

この場合は次のように分ける。

```txt
<p>        -> 開始タグ
こんにちは -> 内容
</p>       -> 終了タグ

全体      -> p要素
```

正確には「タグを書く」というより、タグで要素を作る。

## HTMLは見た目ではなく構造・意味を持たせる

HTMLは、コンテンツの外観を定義するためではなく、コンテンツを構造化するために使う。
見た目はCSSで調整する。

たとえば、`h1` は「大きくしたい文字」ではなく、ページの主見出しという意味を持つ。

```html
<h1>ページの主見出し</h1>
```

大きさを変えたい場合はCSS側で指定する。

```css
h1 {
  font-size: 32px;
}
```

## 属性は開始タグに書く

属性は、要素に追加情報・機能・参照先を与えるもの。

```html
<img src="image.jpg" alt="説明">
<a href="#section">移動</a>
<input type="text">
```

この場合、

- `src`: 画像ファイルの場所
- `alt`: 代替テキスト
- `href`: リンク先
- `type`: 入力欄の種類

という追加情報を要素に与えている。

属性の詳しい分類や `id` / `class` / `role` / `data-*` の使い分けは、[03_HTML属性の基本](./03_HTML属性の基本.md) で扱う。

## 置換要素は外部オブジェクトやUIに置き換わる

置換要素は、要素の中身が外部オブジェクトやブラウザUIに置き換わって表示される要素。

代表例:

- `img`
- `input`
- `video`
- `iframe`

たとえば次のHTML。

```html
<img src="photo.jpg" alt="">
```

これはHTMLの中に画像そのものが書かれているのではなく、`src` で指定した画像ファイルに置き換わって表示される。

そのため、画像はテキストとは違い、置換要素として扱う。

```css
img {
  width: 100%;
  height: auto;
}
```

## void要素は終了タグを持たない

void要素は、終了タグを持たない要素。

代表例:

- `img`
- `input`
- `br`
- `hr`
- `meta`
- `link`

基本は次のように書く。

```html
<img src="image.jpg" alt="">
<input type="text">
```

次のようには書かない。

```html
<img></img>
<input></input>
```

`img` は void要素でもあり、置換要素でもある。
ただし、この2つは別の分類。

- void要素: 終了タグを持たない
- 置換要素: 外部オブジェクトやUIに置き換わって表示される

## ブラウザが暗黙で要素を補うことがある

書いたHTMLと、ブラウザが作ったDOMは完全に同じとは限らない。

たとえば、`tbody` を書いていない場合でも、ブラウザがDOM上で補うことがある。

```html
<table>
  <tr>
    <td>内容</td>
  </tr>
</table>
```

DevTools上では、次のように見えることがある。

```html
<table>
  <tbody>
    <tr>
      <td>内容</td>
    </tr>
  </tbody>
</table>
```

つまり、次の順で考える。

```txt
書いたHTML
-> ブラウザが解析する
-> 必要に応じて補正する
-> DOMになる
```

## DOMはJavaScriptから操作できる構造

DOMは、HTMLドキュメントの構造とコンテンツをブラウザが扱える形にしたデータ表現。

JavaScriptの次の処理は、HTML文字列を直接見ているのではない。

```js
document.querySelector(".button");
```

ブラウザが作ったDOMツリーから、条件に合う要素を探している。

## 認識

- HTMLは、ブラウザに解析されてDOMツリーになる。
- DOMツリーには、要素ノードとテキストノードがある。
- タグは要素を作るための印で、要素そのものとは少し違う。
- 属性は、要素へ追加情報・機能・参照先を与える。
- HTMLは見た目ではなく、構造と意味を担当する。
- 置換要素とvoid要素は別の分類として見る。
