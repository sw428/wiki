# 00_HTML・CSS整理マップ

## 使い方

- 迷ったらまずこのページで置き場所を決める
- 1ファイル1テーマを維持し、重複はリンクでつなぐ
- 初期認識の揺れは各ファイルの「初期認識メモ（確度）」で管理する

## 認知アプローチ（知識に至る手順）

「正解を暗記する」のではなく、次の順で認知を積み上げる。

1. 現象を言語化する
「何がズレたか」「どこで破綻したか」を見た目で書き出す。
2. 基準箱を特定する
font box / [行ボックス](../../03_参照元/行ボックス.md) / border box / flex item box のどれが効いているか切り分ける。
3. 時系列で分解する
[DOM](../../03_参照元/DOM.md) / [CSSOM](../../03_参照元/CSSOM.md) → Layout → Paint → 画像確定 → 再Layout のどこで変化したかを見る（[状態変化](../../03_参照元/状態変化.md)）。
4. 変更対象を分離する
核（コンポーネント）と文脈（レイアウト）を分け、触ってよい層だけ触る。
5. 仮説を1つだけ試す
`width/height`、`aspect-ratio`、`min-width: 0` などを1つずつ当てて差分確認する。
6. 判断理由を残す
「なぜ採用したか」を短く残し、次回の再学習コストを下げる。

## ベース原理セット（2026-05-02 追記）

対話ログで再現性が高かった判断軸を、入口に固定する。

1. 見た目ではなく役割で決める
   要素名・クラス名・構造は「何をしているか」で決める。
2. 外側と内側を分ける
   外側（配置）と内側（整列/見え方）を混ぜない。
3. 構造と余白を分ける  
   `grid/flex` は構造、`margin/gap/padding` は間隔として扱う。
4. 親を先に疑う  
   子がズレたときは、まず親の `display` / 幅 / 既存余白を確認する。
5. 全体ルールを先に作る  
   先に全体、あとで例外（例: `grid-column: 1 / -1`）の順で作る。
6. 表示は要素でなく条件で切る  
   `span` / `div` より「SP/PCどちらで存在させるか」を先に決める。
7. 知っていると使えるを分ける  
   用語理解より「条件付きで再現できるか」を到達基準にする。

## 到達度の見方（2026-06-05 追記）

- このフォルダは、HTML/CSS仕様全体のリファレンスではなく、制作で事故りやすい実用中核を判断ログとして再構成する場所。
- 到達度は「知識をどれだけ知っているか」だけでなく、「既存構造を壊さず、条件 / 理由 / 例外を見て運用できるか」で見る。
- 現時点で強い領域は、ボックス、`display`、インライン、Grid/Flex、画像、背景、メディア比率、セマンティックHTML、BEM、責務分離。
- 追加で厚くする候補は、カスケード、詳細度、継承、セレクタ、疑似クラス、フォーム、アクセシビリティ、CSS変数、`@layer`。
