# 01_テーマ作成手順

## 目的

WordPress制作でテーマを作るときの作業順を残す。

## 最初に見る結論

テーマ制作は、テーマフォルダを作り、必要ファイルを置き、管理画面で有効化して確認する流れ。

自作テーマがまだなくても、LocalでWordPress本体を作成済みなら管理画面にはログインできる。

WordPress本体を作ることと、自作テーマを作ることは分けて見る。

## 確認順

1. LocalでWordPress本体を作る
2. 管理画面へ入れることを確認する
3. テーマフォルダを作る
4. `style.css` を作る
5. `index.php` を作る
6. 必要に応じて `functions.php` を作る
7. 管理画面でテーマが見えるか確認する
8. テーマを有効化する
9. ブラウザで表示を確認する

## CodeJump回ごとの作業判定

### ④ディレクトリ作成編

ここは実際に作るものがある回。

作る場所:

```text
wp-content/themes/blog/
```

最低限作るファイル:

```text
blog/
├── index.php
└── style.css
```

`style.css` にはテーマ名を書く。

```css
@charset "UTF-8";
/*
Theme Name: blog
*/
```

`index.php` には、最初の表示確認用に `Hello World` などを一回だけ書いてよい。

確認後、`Hello World` は消してよい。

## 名前をそろえるとき

WordPressでは、似た名前を分けて見る。

- Localのサイト名: Localアプリ上に出る名前
- ローカルURL: `a-blog.local` など
- PC上の親フォルダ名: Localが管理している実フォルダ
- WordPressサイトタイトル: 管理画面の `設定 > 一般 > サイトのタイトル`
- テーマフォルダ名: `wp-content/themes/blog/`
- テーマ表示名: `style.css` の `Theme Name`

`bloginfo('name')` で表示されるのは、主に管理画面側のWordPressサイトタイトル。

管理画面の `外観 > テーマ` に出る名前は、`style.css` の `Theme Name`。

Localのサイト名、ローカルURL、PC上の親フォルダ名は、wp-adminでサイトタイトルを変えても自動では変わらない。

全部を近い名前にそろえたい場合は、次のように分けて確認する。

```text
Localのサイト名: blog
WordPressサイトタイトル: blog
テーマフォルダ名: blog
Theme Name: blog
```

古いテーマフォルダを消す場合は、先に新しいテーマを有効化して表示確認してからにする。

## Localの親フォルダ名を変えたいとき

Localが管理しているPC上の親フォルダ名は、エクスプローラーで直接変えない方が安全。

Localが覚えているパスと実際の場所がずれると、502 Bad Gateway のように、サーバー側がPHP / WordPressから正しい応答を受け取れない状態になることがある。

名前やURLを整えたい場合は、親フォルダ名を無理に直すより、Localで新規サイトを作ってテーマフォルダだけコピーする方が安全。

流れ:

1. Localで新規サイトを作る
2. 新しいWordPressが起動し、管理画面へ入れることを確認する
3. 古い環境から `app/public/wp-content/themes/blog` をコピーする
4. 新しい環境の `app/public/wp-content/themes/` に貼る
5. `外観 > テーマ` で `blog` を有効化する
6. `設定 > 一般 > サイトのタイトル` を必要な名前へ変える

### ⑤パーツ構成編

ここは、本格的な中身を書くというより、WordPressテーマが複数ファイルで組み立てられることを確認する回。

ただし、まだ作っていないなら、次のファイルを空に近い状態で作っておく。

```text
blog/
├── header.php
├── footer.php
├── sidebar.php
└── index.php
```

中身の実装はあとでよい。まずは「共通部分を分けて読み込む」という構成を作る。

## 自分の頭に残すこと

- まずテーマフォルダ。
- 自作テーマがなくても、WordPress本体ができていれば管理画面には入れる。
- 最低限 `style.css` と `index.php`。
- サイト名、テーマフォルダ名、`Theme Name` は別物。
- Localのサイト名、ローカルURL、PC上の親フォルダ名、WordPressサイトタイトルも別物。
- Localの親フォルダ名は、エクスプローラーで直接変えない方が安全。
- ④は実際にテーマフォルダと最低限ファイルを作る回。
- ⑤は構成確認寄り。ただし未作成なら `header.php` / `footer.php` / `sidebar.php` を作る。
- 最後に管理画面で有効化する。
