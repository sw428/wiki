# 01_WordPress学習の優先順位

## 目的

WordPressを、PHP文法の暗記ではなく、実務で安全に扱うための環境理解として整理する。

## 最初に見る結論

WordPressでは、知識を一貫した体系として全部覚えるよりも、まず次を優先する。

- どのファイルが何を担当しているか
- 触ってよい範囲と、触らない方がよい範囲はどこか
- HTML / CSS / PHP / 管理画面 / Local環境のどこで問題が起きているか
- 変更した結果、何が変わったかを追えるか
- 仕事に直結する型を、間違えずに使えるか

WordPressは、言語そのものを学ぶ教材というより、HTML / CSSで作ったものをテーマ構造へ組み込む環境として見る。

## HTML/CSSや資格学習との違い

HTML/CSSでは、ボックスモデル、レイアウト、余白、レスポンシブなどの基礎概念が重要になる。

CSSは状況判断が多い。

- 親に指定するか、子に指定するか
- `flex` か `grid` か
- `gap` か `margin` か
- 幅を固定するか、流動させるか
- カンプのどこを再現対象にするか

そのため、CSSではスニペットでプロパティを出すだけでは足りない。前提条件と構造判断が先に必要になる。

資格学習では、問題文の意図や出題者が確認したい知識を読むことが重要になる。

一方でWordPressでは、HTML、CSS、PHP、テンプレートファイル、管理画面、テーマ、プラグイン、Local環境、データベースなどが同時に関係する。

そのため、細かい関数を単体で覚えるより、まず「どの層の話か」を切り分けることが重要になる。

## WordPressで先に見る境界線

WordPress学習では、次の境界線を先に見る。

- WordPress本体と自作テーマ
- テーマファイルと管理画面
- `header.php` / `footer.php` / `index.php` / `functions.php` の役割
- HTMLとして書く部分と、PHPで出力する部分
- 固定パスとWordPress関数で作るパス
- CSS/JSファイルそのものと、`functions.php` で読み込む処理
- Local環境で動く部分と、VS Codeで見やすくする補助

原因が分からないまま広い範囲を触ると、表示崩れ、管理画面の不具合、テーマ破損、プラグインとの干渉につながりやすい。

まず変更範囲を小さく保つ。

## 初期学習の順番

最初は次の順でよい。

1. LocalでWordPress本体を作る
2. 管理画面へ入れることを確認する
3. `wp-content/themes/自分のテーマ名/` にテーマフォルダを作る
4. `style.css` と `index.php` でテーマを認識させる
5. `header.php` / `footer.php` / `index.php` へ分割する
6. 固定の画像パスやリンクをWordPress関数へ置き換える
7. `functions.php` でCSS/JSを読み込む
8. 必要になったら、`if` / 変数 / ループ / テンプレート階層へ進む

最初から `WP_Query`、カスタム投稿、カスタムフィールド、プラグイン開発まで広げなくてよい。

## スニペットの位置づけ

WordPress初期では、スニペット活用はかなり重要。

WordPressには、決まった型を正しい場所に置く作業が多い。

```php
<?php get_header(); ?>
<?php get_footer(); ?>
<?php wp_head(); ?>
<?php wp_footer(); ?>
<?php body_class(); ?>
<?php wp_body_open(); ?>
<?php bloginfo('name'); ?>
<?php echo esc_url(home_url('/')); ?>
<?php echo esc_url(get_theme_file_uri('img/common/logo.svg')); ?>
```

これらは、毎回ゼロから想像して書くより、スニペットで正確に出して、何のためにあるかを読む方が実務的。

スニペットは暗記の代替ではなく、認知負荷を下げて、テンプレート構造・変更範囲・原因の切り分けに集中するための補助として扱う。

## CSSとWordPressでのスニペットの違い

CSSでは、プロパティ単位の組み合わせや状況判断が多いため、スニペットより前提条件の把握が重要になる。

WordPressでは、決まったテンプレート構造に組み込み、決められた関数やファイル構成を正しく使うことが重要になる。

そのため、WordPressのスニペットは「楽をするため」だけではなく、余計な場所を触らず、決まった型を正しい場所へ置くための安全装置として見る。

## 自分の頭に残すこと

- WordPressは、言語理解よりも環境理解・境界線・優先順位・リスク管理が先。
- PHPを0から自由に書く力より、まずWordPressの決まった型を読んで正しく置けることが重要。
- HTMLは構造、CSSは状況判断、jsは型変換、状態の受け渡し、WordPressは型・配置場所・責任範囲の切り分けが重要。
- スニペットはズルではなく、壊さず正確に作業するための道具。
